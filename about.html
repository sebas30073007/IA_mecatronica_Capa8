¿Qué es?

CAPA 8

CAPA 8 es una herramienta educativa para aprender redes mediante una IA conversacional que entiende la topología activa y un simulador visual de diagramas. Permite diseñar redes, consultar errores, simular comandos y compartir topologías desde el navegador.

Contexto

El problema

Aprender redes suele ser complejo porque los conceptos son abstractos y las herramientas tradicionales requieren instalación, configuración y conocimientos previos. Packet Tracer y GNS3 tienen una curva de entrada alta.

Además, los chatbots genéricos no entienden el diagrama específico que el estudiante está construyendo: responden de forma genérica, sin contexto de tu topología.

Propuesta

La solución

CAPA 8 combina un editor visual de topologías con una IA que recibe el contexto del grafo activo: nodos, enlaces, IPs, estados y problemas detectados.

El asistente serializa el estado del diagrama y lo inyecta al prompt — el LLM ve la topología real del usuario, no un escenario inventado. Sin instalación, comparte con una URL.

Flujo

¿Cómo funciona?

Usuario diseña la red
La app construye el grafo
La IA recibe contexto de nodos, enlaces e IPs
El sistema clasifica la intención
La IA responde, diagnostica o propone acciones
La app valida y aplica los cambios

Capacidades del sistema

Chat contextual

La IA conoce tu topología activa y responde en función de ella, no de forma genérica.

Editor de diagramas

Canvas SVG interactivo. Arrastra nodos, dibuja enlaces, zoom y pan. 10 tipos de dispositivo.

Terminal virtual

ping, traceroute, ipconfig, show arp, show interfaces.

Pretty layout

Organización jerárquica automática con scoring para elegir la mejor variante visual.

Exportar por URL

Cualquier topología se codifica en Base64 en el parámetro ?g=. Compartir es copiar el link.

Análisis automático

Detecta IPs duplicadas, nodos aislados, falta de gateway y otros tipos de problema.

Stack

Tecnologías utilizadas

Node.js + Express

Backend mínimo: rutas /api/chat y /api/health. Sin base de datos.

Vanilla ES6

Módulos nativos, sin bundler ni framework de UI. Todo el estado vive en un store Redux-like.

Groq API

LLM en la nube. Modelo Llama 3.3 70B. Latencia baja, respuestas en español.

Ollama (local)

Alternativa sin internet. Soporta Qwen3, Gemma3 y otros modelos locales.

GitHub Pages

Frontend estático desplegado. El backend corre separado en Render.com.

SVG Canvas

Nodos como divs absolutos, enlaces como líneas SVG. Zoom/pan con CSS transform.

marked.js Font Awesome 6 CSS glassmorphism sessionStorage Base64 URL codec requestAnimationFrame
